Stitching Photo Mosaics

Background

Given disjoint images of the same scene, we’d like to figure out a way to combine images to develop “stitched” version of the scene that forms a mosaic or panoramic.

In an ideal world, we’d like these photos to be aligned on the same x-y-z rotation axis and completely disjoint, so we can just place the photos next to each other (and maybe apply some blur) to get back the complete scene.

However, capturing full scenes inherently involves rotations (the world is round and we live in 3D) and as such, alignment is not so simple. As such, we make use of homography, where we figure out a linear operation TTT﻿ that maps between common features of the image. Our task, then, is recovering the matrix of transformation. Since (x,y)(x,y)(x,y)﻿ and (x′,y′)(x',y')(x′,y′)﻿ are known, we can expand our system of equations and write the elements of the matrix in terms of the known coordinates. We can then solve the system with least-squares, given that it is overdetermined (number of points is at least 4).

Part B: Automatic Matching

Picking points by hand is very tedious! To fix this, we use automatic corner detection to do this matching. The first part of this paper is based on the paper  “Multi-Image Matching using Multi-Scale Oriented Patches” by Brown et al. 

Harris Corners and Adaptive Non-Maximal Suppression

To start, we label the “corner” features of an image — these are places where 2 edges seem to connect (ie: a corner), which can be detected by changes in brightness. The actual corners identified are the maxima of the Harris Corner Matrix, which is used to detect these changes in brightness. The Harris corners for the left Mainstacks message are depicted below:

However, as we can observe above, there’s a bit of a problem. If we take all the Harris points, they cover the entire image! We’d like to select points to end up using in our homography that are well spaced and form “good descriptors” of the space. To do this, we use adaptive non-maximal suppression. The idea here is that for a given point, we assign a suppression radius — the distance to a stronger corner. Then, we take all the points of a given radius. Feature Detection

We now have the ANMS for both pairs of images — but we need to figure out how to match one set of points to the other. To do this, we start by extracting a 40x40 neighborhood around an image and then apply a Gaussian Blur and downsampling, extracting the core structure in a 8x8 feature region. We the then whiten the this region (set mean to 0 and standard deviation to 1).

Feature Matching

To match images, we first find the feature descriptors of each point. Then, for each point in the first list, we find the first and second nearest neighbor, according to MSE loss. If the ratio of the first and second nearest neighbor: 1NN2NN<t\frac{1_{NN}}{2_{NN}} < t2NN​1NN​​<t﻿, for some threshold ttt﻿, we select the point. The logic of ratio of nearest neighbor is to reduce false positives; we’d like the first nearest neighbor to be signficiantly closer than the second one. Ransac

Now that we have points, we’d like to select and compute homographies like earlier. However, how do we select these points, knowing that there are slight probabilities of points either being false positives or slight mismatches from the true corners. To solve this, we use the RANSAC algorithm:

  1. Select 4 points
  1. Compute the Homography HHH﻿
  1. Compute inlier where the distance between p2p_2p2​﻿ and Hp1Hp_1Hp1​﻿ is less than some ϵ\epsilonϵ﻿
  1. Save the Homography matrix with the the most inlier
